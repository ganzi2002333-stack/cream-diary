### Phase 3 - 数据层
- [x] 创建 `src/js/storage.js` - IndexedDB 完整封装（~400 行）
  - **records 表 CRUD**：saveRecord、getRecordByDate、getRecordById、getAllRecords、deleteRecord、getRecordCount
  - **tags 表 CRUD**：saveTag、getAllTags、deleteTag
  - **settings 表 CRUD**：saveSetting、getSetting、getAllSettings
  - **游标分页查询**：支持日期范围、分数范围、标签、关键词多维度过滤
  - **数据导出**：exportAllData → 完整 JSON（records + tags + settings）
  - **数据导入**：importData → 合并模式（不覆盖已有记录）
  - **清空数据**：clearAllData（危险操作，预留）
  - **3 个 Object Store**：records（keyPath: id, 索引: date/score/tags）、tags（keyPath: id, 索引: name）、settings（keyPath: key）
- [x] 集成 `index.html`：`<script src="js/storage.js">` 在 `app.js` 之前加载
- [x] 更新 `app.js` 集成真实数据层：
  - ✅ **保存按钮** → `CreamStorage.saveRecord()` + 重复记录覆盖确认
  - ✅ **导出按钮** → `CreamStorage.exportAllData()` → Blob 下载 JSON
  - ✅ **导入按钮** → FileReader → `CreamStorage.importData()` → 合并确认
  - ✅ **新建标签** → 同步写入 `CreamStorage.saveTag()`
  - ✅ **提醒开关/时间** → 从 IndexedDB 加载初始值 + change 事件持久化

### Phase 7 - 每日提醒通知
- [x] `app.js` 通知系统：
  - **requestNotifyPermission()**：Web Notification 权限请求（首次弹窗）
  - **sendNotify(title, body)**：发送系统通知（🍰 奶油日记 · 今日提醒）
  - **isTodayRecorded()**：检查今天是否已有记录（有则跳过）
  - **tryNotify()**：五点检查链（开关→权限→时间匹配→今日未记录→发送）
  - **startNotifyScheduler() / stopNotifyScheduler()**：每 60s 轮询调度
  - **initNotifySystem()**：启动时请求权限，授权后启动调度器
  - **开关联动**：开启时重新请求权限+启动调度，关闭时停止调度器
  - **时间匹配**：读取 settings 中的 reminderTime（默认 21:00），精确到分钟匹配
